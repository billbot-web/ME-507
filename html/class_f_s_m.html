◆ run_curstate()

void FSM::run_curstate ( )
inlinenoexcept

Provide the table of states and its length.

Parameters
statesPointer to an array of State* (not owned)
countNumber of entries in the array

The FSM does not take ownership; ensure the pointed-to States outlive the FSM.

Execute the current state's behavior (initializing to the first entry on startup).

Selection rule:

  • This FSM is execute-driven (no predicate-based scanning).
  • On creation it is initialized with a state table; the first update will initialize the current state to the first entry in that table.
  • Thereafter run_curstate() calls the current state's execute() and honors any returned next-state ID to perform a forced transition.

Note: This function does not return the State*; use get_curstate() to query the currently active state when needed.
